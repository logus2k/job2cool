# job2cool — Project Plan v2 (authoritative current state)

> HR assistant ("Diana") that turns a plain-language hiring need into a pack of
> documents — Job Offer, Technical Interviews, Onboarding Plan, Cultural & Team
> Fit — generated by chat, grounded in company KB (vector+graph RAG), written
> live into editable docs, with click-through citations to source PDFs.
>
> This v2 supersedes `project_plan.md` (v1 has the discovery/decision history;
> §9–§14 there are still useful cited references to noted/cv internals).

---

## 1. Architecture (as built)

**Pattern:** a custom job2cool **frontend** + a custom **`job2cool-backend`**
(the "α adapter") that orchestrates the HR flow over **noted's shared services**.
The cv Assistant widget (Diana) is embedded for the chat; the full noted shell
was NOT adopted (it's preserved at `frontend/shell.html` but unused).

```
Browser (logus2k.com/job2cool or localhost:4920)
  ├─ frontend/index.html   custom UI: tabs + split doc/PDF + settings (gold theme)
  ├─ widget/cv-chat.js     Diana: chat + Thinking/Graph/Score/citations + STT/TTS/avatar
  └─ js2c/pdfcite.js       pdf.js citation→PDF+bbox renderer
        │ (same origin)
        ▼
  job2cool-backend  (FastAPI, container `job2cool-backend`, host :4920)
        ├─ owns: /api/chat (SSE), /api/buffers/* (live docs SSE), /api/citation,
        │        /api/graph_trace, /api/score_answer
        ├─ reverse-proxies every OTHER /api/* → noted:8123 (KB/Explorer/doc files)
        └─ calls services on noted-network + logus2k_network:
             agent_server:7701  (gemma-4 + ma2-360m-dpo-b01; preset cv_query_rewriter)
             noted-rag:8200      (/search_multi, /collections, /embed)
             noted-graph:5523    (/research/{domain}/retrieve, /research/{domain}/chunk/{tag})
             noted:8123          (proxied KB/document APIs)
  STT/TTS/avatar: stt_server:2700, tts_server:7700, avatar_server:7800
     (logus2k_network) — reached via /stt,/tts,/avatar on the PROXY origin, NOT
     job2cool-backend. So audio works on logus2k.com/job2cool, not localhost:4920.
```

**Networks:** job2cool-backend joins **`noted-network`** + **`logus2k_network`**
(both external). **Copy-only** guardrail (build from copies; don't modify noted/cv)
— ONE authorized exception so far: edited noted's `frontend/css/panels.css`
(commented `overflow:hidden` on `.jsPanel-hdr`) and rebuilt+relaunched the
`noted` service (owner-requested).

## 2. Models & KB

- **gemma-4** — orchestrator/composer (vision+reasoning, 128K ctx; returns
  reasoning as inline `<think>` in `content`). `JOB2COOL_GEMMA_MODEL`.
- **ma2-360m-dpo-b01** — specialist Job-Offer drafter (SmolLM-360M DPO). NL need
  in → Markdown offer out. `JOB2COOL_DPO_MODEL`.
- Judge = gemma JSON (the `cv_rag_judge` preset 400s on this stack, so we judge
  with gemma + an explicit JSON instruction). `JOB2COOL_JUDGE` (default gemma).
- Query rewriter = `cv_query_rewriter` preset (reused). `JOB2COOL_QUERY_REWRITER`.
- **KB = role-aware onboarding domains** `jobs_onboard_<family>`: devops,
  architect, backend, data_eng, embedded, mobile, frontend, general, ml_ai, qa,
  security. **Only `jobs_onboard_devops` + `jobs_onboard_architect` exist today**;
  owner is creating the rest. Per turn gemma classifies the role →
  `jobs_onboard_<family>`, with graceful fallback (general → any existing
  jobs_onboard_*). **Other-section domains** (interview banks, culture, benefits)
  are NOT yet provided — all sections currently ground on the resolved onboarding
  domain; section-specific domains come later (owner).

## 3. Backend files (`job2cool/backend/`)

- **main.py** — FastAPI. Serves `frontend/` static (`/static`, `/wallpapers`,
  `index.html` at `/`). Owns: `/api/health`; `/api/chat` (POST→`orchestrator.run_chat`,
  SSE); `/api/buffers/events/stream` (SSE of doc changes), `/api/buffers/{id}/save`;
  `/api/citation/{tag}` (cache + noted-graph `/chunk` → source/page/bbox/regions);
  `/api/graph_trace` (last turn's entities/edges); `/api/score_answer` (gemma JSON
  judge over cached turn). Catch-all `/api/{path}` reverse-proxies to noted:8123
  for everything not owned (`_OWNED_API_PREFIXES`).
- **orchestrator.py** — the HR pipeline. `run_chat(message, history, config)`:
  (0) `_resolve_need` rewrites the message using conversation history (memory!);
  (1) `_role_label` → role title or "" → if "" **ask which position, generate
  nothing**; (2) `resolve_onboard_domain`; (3) `_requested_sections` (LLM
  classifies which of offer/interview/onboarding/culture, or all) → only those
  deliverables; intro stream (`INTRO_SYSTEM`, Diana persona) + a `<voice>` line
  (avatar speaks only this); per section: lazy-create a buffer (=tab), aggregated
  RAG, MA2 draft (offer, if `ma2` in `config.offer_sources`) → gemma refine/
  compose (`SECTION_MAX=8192`), write to buffer with a clickable `**Sources:**
  [markdown_chunk:hex]` footer (`_cited_sources`); cited summary; `meta`
  (turn_id, deliverables, offer_ma2, role, domain, counts). `config.offer_sources`
  = subset of {ma2,gemma,rag}; when both ma2+gemma, also writes a `Job Offer (MA2)`
  buffer for the UI toggle.
- **services.py** — agent_server LLM (`llm_complete(...,think=bool)`,
  `llm_stream`); RAG: `formulate_query`, `search_multi`, `graph_retrieve`,
  `graph_and_vector_search` (aggregated vector+graph), `resolve_chunk_regions`
  (noted-graph `/chunk` → page/bbox); role domain: `available_corpus_domains`,
  `classify_role_family`, `resolve_onboard_domain`. **`think=False` on all utility
  passes** (role/classify/rewrite/summary/judge) — else gemma spends the budget in
  `<think>` and returns empty.
- **buffers.py** — in-memory live-doc buffers + asyncio pub/sub → SSE. One buffer
  per deliverable (name = deliverable title). `DocBuffer{buffer_id,name,content}`.
- **cache.py** — chunk cache (by citation hex), turn cache (question/evidence/
  answer/entities/edges/domains), `last_turn()`.
- **Dockerfile** (`COPY *.py`), **requirements.txt** (fastapi/uvicorn/httpx/pydantic).

## 4. Frontend files (`job2cool/frontend/`)

- **index.html** — the UI. Header (logo `images/job2cool_logo_small.png` @56px,
  "job2cool", **💬 Assistant** + **⚙ Settings** buttons; no subtitle/badges).
  **Tab bar** (one tab per generated deliverable, on-demand, auto-focused). Doc
  area = flex split: **`#doccol`** (rendered Markdown + clickable `.cite` badges)
  **|** **`#pdfcol`** (PDF split pane, hidden until a citation opens it). Settings
  modal (Job Offer Generation: MA2/Gemma/RAG → `window.JOB2COOL_CONFIG`). Per-tab
  **Copy MD** + **PDF export** (print-in-place via `#printarea` + `window.print()`,
  no new tab). Job Offer tab has a **Gemma/MA2 segmented toggle** when both exist.
  Gold accent `--primary:#ffe19b` (dark text on gold). Globals: `JOB2COOL_CONFIG`,
  `JOB2COOL_NEW_TURN` (reset tabs on send), `JOB2COOL_OPEN_PDF(data)` (open split
  pane + `JOB2COOL_RENDER_PDF`). cv-chat panel width `--cvchat-panel-w:
  clamp(33vw,600px,50vw)` (1/3→1/2; mobile 100vw via cv-chat.css). `.cvchat-launcher`
  hidden (open via header button).
- **widget/cv-chat.js** — cv's Diana widget, PATCHED (keep this list!):
  (a) `/chat` POST body adds `config: window.JOB2COOL_CONFIG`; (b) calls
  `window.JOB2COOL_NEW_TURN()` on send; (c) citation handler: a chunk with
  `source_path` → `window.JOB2COOL_OPEN_PDF(data)` (split pane) instead of the old
  in-page highlight; (d) asset paths resolved via `new URL(p, document.baseURI)`
  for three.js import + worklet + VAD/onnxruntime (sub-path safe); (e) persona =
  **Diana / "HR Assistant"**, `diana.png`, 20 SHORT random greetings
  ("I'm Diana, your HR Assistant. Tell me what you need."), short disclaimer.
  Talks to `/api/chat|citation|graph_trace|score_answer` + `/stt,/tts,/avatar`
  socket.io (origin).
- **widget/cv-chat.css**, **widget/diana.png**, **widget/recorder-worklet.js**,
  **widget/vendor/** (onnxruntime-web, vad, three, jspanel, marked, katex,
  highlight, socket.io).
- **js2c/pdfcite.js** — `JOB2COOL_RENDER_PDF(body,data)`: pdf.js render of cited
  page(s) + bbox (`convertToViewportRectangle`), **ResizeObserver auto-fit** to
  the pane width (re-renders, bbox stays aligned). `JOB2COOL_PDF_URL`. pdf.js from
  `static/vendor/pdf.min.mjs` (+worker+wasm, from the noted shell copy).
- **vendor_j2c/marked.min.js** (doc Markdown). **vendor/**, **css/**, **js/**,
  **shell.html** = the copied noted shell (mostly unused; pdf.js + wallpapers live
  under vendor/). **images/job2cool_logo_small.png**.
- Frontend is **bind-mounted** into the container (`./frontend:/app/frontend:ro`),
  so edits are live — NO rebuild for frontend changes (backend changes need
  `docker compose up -d --build`).

## 8. Gotchas / decisions to remember

- `think=False` for utility LLM calls (empty output otherwise).
- Frontend served under a SUB-PATH (`/job2cool/`) via the proxy → all dynamic
  asset URLs must be **base-relative** (`document.baseURI`), never `/static/...`.
- Citations resolve to bbox only for **graph excerpts** (`markdown_chunk:hex`);
  dense search chunks have no bbox (open PDF without box).
- cv-chat.js is PATCHED in ~5 places (§4) — re-applying a fresh cv-chat copy would
  lose them.
- Segregation & memory are **LLM-decided** (no regex/keywords) — see §3
  orchestrator.
- Audio routing (`/stt,/tts,/avatar`) is the existing proxy's job, not
  job2cool-backend ("don't touch nginx").
